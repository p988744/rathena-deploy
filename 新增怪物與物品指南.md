## 新增怪物

### 怪物方案 A：複用現有 Sprite

**原理：** 客戶端透過 `NPCIdentity.lub` 將怪物 ID 對應到 sprite 名稱，再去 GRF 裡找對應的 `.act`/`.spr` 檔案。複用時只需讓新 ID 對應到已存在的 sprite 名稱即可。

#### Step 1：服務端 `server/db/re/mob_db.yml`

新增（或取消 comment）一筆怪物資料：

```yaml
  - Id: 99999
    AegisName: MY_CUSTOM_MOB
    Name: My Custom Mob
    Level: 50
    Hp: 10000
    BaseExp: 5000
    JobExp: 3000
    Attack: 300
    Attack2: 500
    Defense: 20
    MagicDefense: 10
    Str: 50
    Agi: 30
    Vit: 20
    Int: 15
    Dex: 40
    Luk: 10
    AttackRange: 1
    SkillRange: 10
    ChaseRange: 12
    Size: Medium
    Race: Demi_Human
    Element: Neutral
    ElementLevel: 1
    WalkSpeed: 200
    AttackDelay: 1000
    AttackMotion: 480
    DamageMotion: 360
    Ai: 21
    Drops:
      - Item: Apple
        Rate: 5000
```

常用 `Ai` 行為值：

| 值 | 行為 |
|----|------|
| `02` | 被動（不主動攻擊，Poring 風格）|
| `04` | 主動攻擊 |
| `21` | 主動攻擊 + 輔助同伴 |

#### Step 2：客戶端 `NPCIdentity.lub`

路徑：`client/data/luafiles514/lua files/datainfo/NPCIdentity.lub`

在 `jobtbl = {` 區塊內新增，**sprite 名稱必須對應 GRF 裡已存在的 `.act`/`.spr` 檔名**：

```lua
jobtbl = {
    -- ... 現有內容 ...

    -- 複用現有 sprite：JT 名稱改成已有的 sprite 名稱
    JT_MY_CUSTOM_MOB = 99999,
    -- 注意：客戶端會用 "MY_CUSTOM_MOB" 這個字串去 GRF 找 sprite
    -- 如果 GRF 裡沒有這個名稱的檔案，怪物會顯示異常
    -- 最簡單的複用方式：讓名稱對應到已知存在的 sprite
}
```

**最省事的複用技巧**：直接用已有怪物的 ID（例如讓自訂怪物顯示成 Poring），只改 `mob_db.yml` 的 stats，ID 沿用現有的，不需要改 `NPCIdentity.lub`。

### 怪物方案 B：新增全新 Sprite

#### 額外需要的步驟

**Step 1：準備 sprite 檔案**

每個怪物 sprite 需要以下兩個檔案（包含所有方向與動作幀）：

```
data/sprite/몬스터/MY_CUSTOM_MOB.act    ← 動作定義
data/sprite/몬스터/MY_CUSTOM_MOB.spr    ← 圖片資料
```

> `몬스터` 是韓文「monster」，GRF 路徑固定為此。

**Step 2：打包進 GRF**

使用 **GRF Editor**（Windows 工具）：
1. 開啟 `data.grf` 或建立新的 `custom.grf`
2. 將 `.act` 與 `.spr` 加入對應路徑
3. 若使用獨立 custom GRF，在 client 目錄的 `data.ini` 設定讀取順序（數字越小越優先）：
   ```ini
   [Data]
   0=custom.grf
   1=data.grf
   ```

**Step 3：`NPCIdentity.lub` 新增對應**

```lua
JT_MY_CUSTOM_MOB = 99999,
```

> JT 名稱（去掉 `JT_` 前綴後）必須與 sprite 檔名完全一致（大小寫敏感）。

## 新增物品

### 物品方案 A：複用現有 Sprite

**原理：** `AliasName` 欄位告訴 server 回傳另一個已知物品的 AegisName 給 client，client 就會使用那個物品的所有圖示與外觀。

#### Step 1：服務端 item_db

依物品類型選擇對應檔案：

| 類型 | 檔案 |
|------|------|
| 武器、防具 | `server/db/re/item_db_equip.yml` |
| 使用道具、材料 | `server/db/re/item_db_use.yml` |
| 其他雜物 | `server/db/re/item_db_misc.yml` |

範例（武器）：

```yaml
  - Id: 60000
    AegisName: My_Custom_Sword
    Name: My Custom Sword
    AliasName: Sword            # ← 關鍵：讓 client 使用 Sword 的外觀與圖示
    Type: Weapon
    SubType: 1hSword
    Buy: 50000
    Weight: 1000
    Attack: 200
    Range: 1
    Slots: 2
    Jobs:
      Knight: true
      Swordman: true
    Locations:
      Right_Hand: true
    WeaponLevel: 4
    Refineable: true
    Script: |
      bonus bStr,10;
      bonus bAtk,50;
```

範例（消耗品）：

```yaml
  - Id: 60001
    AegisName: My_Custom_Potion
    Name: My Custom Potion
    AliasName: Red_Potion        # 使用紅藥水的圖示
    Type: Usable
    Buy: 1000
    Weight: 30
    Script: |
      percentheal 50,0;
```

#### Step 2：客戶端 `itemInfo_true_org.lub`

路徑：`client/System/itemInfo_true_org.lub`

在檔案末尾（`}` 前）新增：

```lua
[60000] = {
    unidentifiedDisplayName = "Unknown Sword",
    unidentifiedResourceName = "¹«±â",       -- 未鑑定時的 sprite（韓文，可沿用現有）
    unidentifiedDescriptionName = { "An unidentified sword." },
    identifiedDisplayName = "My Custom Sword",
    identifiedResourceName = "¹«±â",          -- 複用現有武器 sprite 名稱
    identifiedDescriptionName = {
        "A powerful custom sword.",
        "^FF0000STR +10, ATK +50^000000",
        "^777777Weight: 100^000000"
    },
    slotCount = 2,
    ClassNum = 1,   -- 裝備外觀 class（1H 劍 = 1）
},
```

常用 `ClassNum` 值（武器外觀分類）：

| ClassNum | 對應外觀 |
|----------|----------|
| 1 | 單手劍 |
| 2 | 雙手劍 |
| 3 | 匕首 |
| 4 | 弓 |
| 5 | 法杖 |

> 繁中名稱也可以改在 `itemInfo_true.lub` 的 `zh_names` 區塊覆蓋，格式為 EUC-KR 編碼字串。

#### 額外需要的步驟

**物品 inventory 圖示：**
```

```
> `À¯ÀúÀÎÅÍÆäÀÌ½º` 是韓文「userinterface」。圖示尺寸為 24x24 px（BMP 格式）。

**裝備在角色身上的 sprite：**
```
data/sprite/¾ÆÀÌÅÛ/MY_WEAPON.act
data/sprite/¾ÆÀÌÅÛ/MY_WEAPON.spr
```
> `¾ÆÀÌÅÛ` 是韓文「item」。

**圖鑑（collection）圖示：**
```
data/sprite/¾ÆÀÌÅÛ/collection/MY_WEAPON.bmp    -- 160x112 px
```

**`itemInfo_true_org.lub` 對應設定：**

```lua
[60002] = {
    unidentifiedDisplayName = "Unknown Item",
    unidentifiedResourceName = "MY_WEAPON",
    unidentifiedDescriptionName = { "" },
    identifiedDisplayName = "My New Weapon",
    identifiedResourceName = "MY_WEAPON",       -- ← 對應新 sprite 檔名（不含副檔名）
    identifiedDescriptionName = {
        "A brand new custom weapon.",
    },
    slotCount = 1,
    ClassNum = 1,
},
```

## 總結比較表

|  | 怪物 - 複用 sprite | 怪物 - 新 sprite | 物品 - 複用 sprite | 物品 - 新 sprite |
|---|---|---|---|---|
| `mob_db.yml` | ✅ 必要 | ✅ 必要 | — | — |
| `item_db_*.yml` | — | — | ✅ 必要（加 `AliasName`）| ✅ 必要 |
| `NPCIdentity.lub` | ✅ 必要 | ✅ 必要 | — | — |
| `itemInfo_true_org.lub` | — | — | ✅ 必要 | ✅ 必要 |
| GRF `.act`/`.spr` | ❌ 不需要 | ✅ 必要 | ❌ 不需要 | ✅ 必要 |
| GRF `.bmp` 圖示 | ❌ 不需要 | ❌ 不需要 | ❌ 不需要 | ✅ 必要 |
| 需 rebuild server | ✅ 是 | ✅ 是 | ✅ 是 | ✅ 是 |
| 需更新客戶端 | ✅ 是（.lub）| ✅ 是（.lub + GRF）| ✅ 是（.lub）| ✅ 是（.lub + GRF）|

## 部署流程提醒

### Server 端變更（yml 修改）

```bash
# 需要 rebuild + redeploy
cd server/
./update.sh
```

### Client 端變更（.lub 或 GRF 修改）

```bash
# 只需要把更新後的檔案讓玩家覆蓋即可
# .lub 檔案直接替換：
client/System/itemInfo_true_org.lub
client/data/luafiles514/lua files/datainfo/NPCIdentity.lub

# GRF 更新則需重新打包並提供新版 GRF 給玩家下載
```

> `.lub` 變更**不需要** rebuild server，只需更新玩家端的對應檔案。
